- `TMath::` 调用TMath库（是命名空间，表示“我要从 ROOT 的数学仓库里拿东西”）
	- `Pi()` 调用圆周率常数
	- `Erf(0.2)` 看测量值距离中心值有多少$\sigma$，算出的结果是随机变量落在这个范围的概率，这个概率越大越好，说明你的测量值不在这个区间内，一般不会出现，不是偶然事件
		- $erf(x) = \frac{2}{\sqrt{\pi}} \int_{0}^{x} e^{-t^2} dt$ 
		- 计算从 $0$ 到 $0.2$ 这个范围下 $e^{-t^2}$ 曲线底下的面积，将面积乘以系数 $\frac{2}{\sqrt{\pi}}$（归一化）
		- 计算物理问题时要用`TMath::Erf(n / sqrt(2))`
			- **数学上的 `Erf(x)`**：定义里的指数是 $e^{-t^2}$。
			- **物理上的正态分布**：指数是 $e^{-x^2 / 2\sigma^2}$。
	- `power(x,i)` x的i次方
	- `Abs()` 取绝对值

-  **`TSpectrum`** 简单来说，`TSpectrum` 是 ROOT 里专门用来处理“谱图”（Spectrum，通常指带有峰的直方图，如能谱、质谱）的高级工具包。它的主要能力包括**寻峰（Peak Search）和本底估算（Background Estimation）**。
	- **`TSpectrum *s = new TSpectrum(500);`**
	    - **初始化工具**：创建了一个 `TSpectrum` 的对象实例，命名为 `s`。
	    - **参数 `500`**：这是告诉工具，在后续的寻峰操作中，最大允许寻找 **500** 个峰。虽然这段代码主要在讲算本底，但这是初始化该类的标准写法。
	- **`s->Background(h0, 15, "same");`**
	    - **核心功能**：调用 `Background` 函数来自动计算直方图的“底色”或“基线”。
	    - **`h0`**：输入的原始直方图（包含了信号峰+本底）。
	    - **`15` (即 `niter`)**：这是**迭代次数**，最关键的参数，`niter` 的数值**越大**，算出来的本底线越**平滑**，位置也越**低** （可以多改几个看看）
	    - **返回值**：函数返回一个新的直方图指针（代码中转为了 `TH1F* hb0`），这个新直方图里存的就是计算出来的**本底曲线**。
	- `Int_t nfound = s->Search(hh, 2, "", 0.005);`
		- - **`s->Search(...)`**：命令 `TSpectrum` 对象 `s` 去直方图 `hh` 里找峰
		    - **`hh`**：要处理的直方图。
		    - **`2` (Sigma)**：**最关键的参数**。这代表你期望的峰的“胖瘦”（标准差）。
		        - 如果你填 `2`，算法就会去找那些宽度大约为 2 个 bin 的峰。
	        - **注意**：如果峰很宽（比如半高宽是 10），这里填 1 或 2，算法就会把一个大峰识别成很多个小毛刺（假峰）。反之亦然。**必须根据实际情况调整这个数字。**
		    - **`""`** (Options)：选项。留空表示默认模式（会把找到的峰用倒三角标记画在图上）。如果填 `"goff"` (Graphics OFF) 则不画图。
		    - **`0.005` (Threshold)**：阈值。
		        - 意思是被选中的峰，高度至少要达到最高峰的 **0.5%**，这是为了过滤掉那些极小的噪音峰。
- `RooRealVar` (实值变量)，它既可以表示观测到的物理量（如粒子质量 $m$、能量 $E$），也可以表示模型中的参数（如高斯分布的均值 $\mu$、标准差 $\sigma$）。
	- **功能：** 定义变量的名称、当前值、允许的变化范围（Min 到 Max）以及单位。
	- **特点：** 如果不固定（`setConstant(true)`），在拟合过程会自动改变以寻找最优解。
	- `RooRealVar x("x", "x coordinate", 0, -10, 10);`
- `RooAbsData` (抽象数据基类)，这是一个抽象基类（Abstract Base Class），代表了**数据集**，通常不会直接使用 `RooAbsData`，而是使用它的子类，最常见的是：
	- **`RooDataSet`**: 未分箱的数据（Unbinned data），即一系列具体的事件点（Event 1: x=1.2; Event 2: x=1.5...）。
	- **`RooDataHist`**: 分箱的数据（Binned data），即直方图。
	- **功能：** 存储实验测量值或蒙特卡洛模拟数据。
	- **关系：** 它通常是被拟合的对象（即：用模型去拟合 `RooAbsData`）。
- `RooWorkspace` (工作空间)，允许将所有的变量（`RooRealVar`）、模型（PDFs）、数据（`RooAbsData`）以及它们之间的复杂的依赖关系全部打包存放在一起。
	- **功能：**
	    - **持久化：** 可以把整个复杂的分析环境存入一个 `.root` 文件中，发给别人，别人打开后能直接用，不需要重新定义变量。
	    - **内存管理：** 它接管了对象的生命周期，你不需要担心 C++ 指针乱飞的问题。
	    - **工厂模式：** 提供了一个简单的 `factory()` 方法来快速创建模型字符串。
	- 比喻： 如果 `RooRealVar` 是螺丝，模型是机器，那 `RooWorkspace` 就是整个车间，里面放好了机器和说明书，关上门（存文件）随时可以搬走。
- `ModelConfig` (模型配置)，当面对一个复杂的 `RooWorkspace` 时，里面可能有几十个参数，这个函数用来明确告诉统计工具（如计算置信区间或发现显著性时）哪些东西是什么。
	- **功能：** 它明确指定了：
	    - **POI (Parameters of Interest):** 最关心的参数（比如信号强度 $\mu$）。
	    - **Nuisance Parameters:** 讨厌但必须处理的系统误差参数（比如探测器效率的不确定性）。
	    - **Observables:** 哪些变量是观测值（比如不变质量）。
	    - **PDF:** 哪个函数是用来描述数据的总概率密度函数。
	- **场景：** 做假设检验（Hypothesis Testing）或计算各种 limit

## 命令行
- **`auto pVec = [](const doubles& v){for (auto&& x:v) cout << x << endl;}`**
	- `auto pVec` 相当于`void main`，只不过auto的意思是后边类型很复杂，电脑自己判断
	-  `[]` —— Lambda 标志
		- 只要看到 `[]` 开头，就意味着“我要开始现场造一个函数了”
		- 方括号里如果是空的，说明这个函数是个“独行侠”，它不需要使用函数外面定义的变量（比如外面定义的 `N` 或 `x`）
	- `(const doubles& v)` —— 参数列表，定义了这个函数接收什么东西作为输入
		- **`doubles`**：这是你在上一行代码里定义的“外号”，实际上就是 `std::vector<double>`（装小数的数组）。
		- **`&` (引用/Reference)**：**这非常重要！**
			-  **不加 `&` (按值传递)**：如果写 `[](doubles v)`，程序会把外面那个巨大的数组完整复印一份塞给 `v`，如果数组有 100 万个数据，内存瞬间爆炸，速度极慢。
			- **加上 `&` (按引用传递)**：如果写 `[](doubles& v)`，程序不会复制数据。它只是给外面的数组起了一个“别名”叫 `v`。函数里的 `v` 就是外面的那个数组本身。零拷贝，速度飞快。
		- **`const`**：只读，保证这个函数只能**看**数组里的数，绝对不能**改**数组里的数。
	- `{ ... }` —— 函数体
		- `for (auto&& x : v)`
			- `:` (冒号)：读作“in”（在...里面），意思就是“对于 v 里面的每一个元素...”
			- `x`：给当前拿到的这个元素起的临时名字
			- `auto&&`：这是个万能写法，简单理解：它告诉编译“用最合适、最高效的方式把元素拿出来给 x”，不管 v 里面是只读的、可写的、还是临时的，`auto&&` 都能自动适配，避免不必要的拷贝。
        - **`cout << x << endl`**：
	        - `cout`：屏幕（控制台）。
	        - `<<`：“流操作符”。想象成一个漏斗或传送带，把 `x` 和`endl` 送到屏幕上
	        - `endl`：End Line（换行符），相当于敲了一下回车
	- &&和&
		- `int&&`：只接受右值，比如 `10`，或者 `x + y` 的计算结果，是临时的，没有名字，马上就会被销毁。
		- `int&`：只接收左值，比如 `int a = 10;` 里的 `a`，有名字有固定的内存地址，不会随便消失。
		- `auto&&`：也能接收左值也能接收右值，且不产生多余的内存复制
