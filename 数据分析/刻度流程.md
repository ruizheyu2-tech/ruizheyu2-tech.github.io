## 流程概述

>root2root->1.1->2.1

root2root会将原文件转成
	 **xxx_tmp.root:** 
     基本保留原文件信息(除了时钟翻转之类的信号)，tree结构按照board区分，数据按照DAQ读取到的时间顺序存入对应的tree
	 **xxx_map.root：**
     按照时间标记（TriggerTimeTag）排序并进行事件重构，同时将不同的通道映射(map)到对应的探测器上。这个也基本上保留原始信息，也是后期其他分析程序的基础。
	 **xxx_chain.root**：
     衰变链的默认搜索结果。如果需要变更条件或者限制更严格的搜索条件，建议在Reconvert_chain那边添加

1.1 过程用来提取各路的单谱数据，2.1生成刻度系数

>->root2root->1.3->2.1

把上一步得到的刻度系数代入root2root再转文件

1.3 过程用来提取[[信号处理#梯形算法]]的刻度单谱文件，2.1生成刻度系数

>->root2root->1.2->2.1

就用上一步转出来的文件，不用再转一遍了

1.2 过程用来提取[[信号处理#波形拟合算法]]的刻度单谱文件，2.1生成刻度系数

>->root2root->2.2->3

把上一步得到的刻度系数代入root2root再转文件

2.2 过程用来内刻，但统计量太小这个自动的找不准，只能自己手动内刻，3生成SSD刻度系数

### map2Hist:提取对应路数的能量单谱

>文件地址：SHANS006/ANA_SHANS2_202509_v1.07.3/Cal_Auto_All_202412/map2hist

这个需要的map文件必须是没有刻度系数的

转完root2root之后的map文件要加在一起`hadd SHANS006000250 SHANS006{021,022,239,245}_map.root`

该程序功能在将多个map文件里面的Si和Ge信号按照通道数转换成每个通道的能量单谱，用于后面的能量刻度。

这里可以是标准源的刻度文件，也可以是正式实验的数据（正式实验只转有效decay信号，并且只有单重点火记录，如果有其它需求在Data_Map.cpp的Analyze_Event函数中修改）对应的通道会生成对应的h1_Exxx的单谱。

**需要操作的部分：**

- setup.h文件对应的map文件的路径OUTPUT_FILE_PATH和文件前缀
	- 在main.cpp中可以找到这里![[Pasted image 20260121100108.png]]
		- `Form`生成了一个文件路径字符串，相当于生成了一个地址条
		- `access(str_tmpl.Data(), F_OK)`这一步就是拿这个地址条去路径里找这个文件，如果文件存在返回0，不存在返回-1
			- 系统函数逻辑（存在返回0，不存在返回-1）`access()`
			- 布尔逻辑（真返回1，假返回0）`if()`
	
- 根据需要设置能谱删选条件，在Data_Map.cpp的Analyze_Event函数中，比如将噪声信号去掉，防止后期自动寻峰失败。
```
//get X energy
if (DSSDX_mul == 1 && DSSDX_E[0] > 1000)
//只有一个通道被击中了，用来排除噪声或多个粒子同时击中的情况，后边的是能量阈值
{
    h1_EI[(int)DSSDX_Ch[0]]->Fill(DSSDX_E[0]);//将该次击中的能量值填入对应通道的直方图中
    data_Cal.EiX = DSSDX_E[0];
    data_Cal.ChX = DSSDX_Ch[0];
}
```

### **AutoCal_Si_Std: 生成alpha标准源/Gamma源的刻度系数**

>文件地址：SHANS006/ANA_SHANS2_202509_v1.07.3/Cal_Auto_All_202412/Cal_Auto_Si_Std_Source_202412

生成对应探测器的刻度文件，使用前检查一下源的设置，在AutoCalibration.cpp中：  

  void AutoCalibration::Set_StdAlpha_Energy()
  void AutoCalibration::Set_StdGamma_Energy()

内的参数，按照原有格式，更改成实际使用的标准源信息。在现在终端情况标准源下可以不用管。

**需要操作的部分**：

  1. 在main.cpp里面找到ptr_Auto->Set_SearchPar(sigma, threshold, Chi)函数, sigma参数为峰的sigma值，需要与实际相近最好， threshold不要太低，防止找到太多的峰，影响匹配
  2. 在main.cpp里面找到ptr_Auto->Set_FitRange(left，right)函数, 峰型拟合范围，数值为百分比，一般根据探测器的分辨设置，但是对于某些峰型有问题的，比如对于alpha源信号，峰型左边有其他峰影响，就少取一点。

**执行格式：**`./AutoCal_Si_Std filename` 

filename为前一步生成的Map_Decay_runnum1_runnum2.root就可以在当前目录下生成刻度文件，ener_cal_Map_Decay_runnum1_runnum2.dat

对于ener_cal_Map_Decay_runnum1_runnum2.dat，从左至右依次为通道、截距、斜率、第三个峰的能量分辨FWHM、三个α峰的拟合方差，注意0-127条DSSDX的能量分辨~20 keV，128-175条DSSDY的能量分辨25-50 keV，176-223条SSD的能量分辨60-90 keV
![[Pasted image 20260121102917.png]]

- 检查刻度系数，如果得到的刻度系数出现问题
	
	- 刻度系数直接是0.0，1.0 (这两个数值说明自动拟合失败，除非极端情况参数就是这个)
	
	- 最后一个数值特别高(拟合方差CHI值，根据不同探测器会不同,特别高可能峰型或者别的问题，需要自己手动检查)
	
	- 第一个参数很大的时候(数字化仪的基线正常应该为比较小的数值，特别大说明有问题)。

这时候需要检查问题，进行debug，首先将setup.h中的 `#DEBUG_MSG_OFFLINE`取消注释，重新编译之后，执行`./AutoCal_Si/Ge filename  debug_ch`

- 就可以查看自动寻峰的结果，以及拟合以及跟标准能量对应的过程，判断出现问题的地方。可能的问题：

	 1. 程序对最高峰的认定很敏感，所以确保寻峰的最高能量的峰对应于实际需要刻度的最高峰，可以在生成histgram图时候做一些限制（去掉低能噪声和过阈信号） 
 
	 2. threshold太低，把一些低能的峰也寻找出来，导致最后的寻峰结果和需要刻度的能量峰的匹配出错
 
	 3. Chi设置太低，丢弃掉很多峰
 
	 4. Sigma不合适，导致寻峰失败，一般这个是主要原因
 
	 5. 设置的拟合范围不合适，比如对于alpha峰，由于有低能峰的影响，拟合范围太宽会导致峰位飘向低能，导致线性拟合参数的第一个参数偏大或者chi方差值偏大

```

29 #define FT_GAP  8      //in 10ns,  
30 #define SL_RS  3       //in 10ns,  
31 #define SL_GAP  3       //in 10ns,

33 #define CFD_F      0.3   //
```

## **Cal_Auto_SSD_RECAL：SSD内刻参数提取**

该程序用于对SSD的信号进行内源刻度。主要思路是：在DSSD完成内源刻度之后，SSD完成alpha源刻度，然后利用alpha衰变链确定alpha1为SSD+DSSD的加和信号来进行刻度。

难点在于:

1. 需要确保每一条SSD上都有一定统计量的加和信号。
2. 由于不同的条的加和能量差异很多，如下图所示，不同条的加和能量差异能到200多keV，因此能量设置的能量范围需要大一点，但不能设置的太大（可以事先看一下不同条大致的能量范围），要不然可能会带入别的峰，如果引入别的峰也需要保证目标峰是最高峰。因为现在程序选取的是最高峰，如果带入更高幅度的能量峰，会导致拟合目标出现问题.![[Pasted image 20260119221556.png]]
图8. 2 焦平面BB17双面硅条探测器（DSSD）
  
原程序中利用二维关联确定了α1，并且利用这个α1进行刻度，在统计量不够高的情况下，可以直接卡一维的条件，在这种情况下需要改Data_Rechain.cpp中的第46行的if条件。 在202311版本的测试使用中发现，新版本如果只用一个峰区做SSD内刻时候，外推的加和信号（比如刻度是7477keV，外推到5304keV的话中心峰位能差异到30+keV），因此现在建议用多个峰同时进行内刻，峰选取宽一点比较好。

1. 在main.cpp设置衰变链条件，在开头部分，设置的条件改成如下结果
```
typedef struct
{
int ID;
int Decay_Chain_Num;
double   Peak_energy;
int   runnum_start;
int   runnum_stop;
std::vector<double>Alpha_E_Left;
std::vector<double>Alpha_E_Right;
std::vector<double>Alpha_DecayT;
}DECAY_INFO_T;
```
2. 在Data_Rechain.cpp里面的Process()函数，这个是加和信号的删选条件，根据具体情况更改
  
该程序需要在程序所在目录下执行`./Auto_Cal_SSD_RECAL num1 num2`

刻度结果存在ener_cal_SSD.dat中，同时会形成一个Cal_SSD_runnum1_runnum2.root文件， 刻度系数文件输出结果前两列是线性刻度系数，第三列是拟合峰FWHM，第4列是拟合的峰位与拟合目标能量peak_energy的差值。

如果觉得刻度系数有问题，可以根据这个文件自己手动刻度或者用debug模式具体看看什么问题。同时文件中会有一个好h1_total的histgram，可以用来查看刻度之后的分辨。
